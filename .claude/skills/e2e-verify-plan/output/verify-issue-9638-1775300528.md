# Verification Guide: #9638 -- Positron Notebooks: Add MVP Outline

**Issue**: https://github.com/posit-dev/positron/issues/9638
**PR**: https://github.com/posit-dev/positron/pull/12404
**Type**: Enhancement (new feature)

## Summary

Adds Outline pane support for Positron notebooks, showing markdown headers (with nesting) and code cell first-line previews. Enables the standard VS Code outline pane, breadcrumbs, and Go to Symbol when a Positron notebook is active.

## Primary Verification

### Scenario 1: Outline shows markdown headers and code cell previews

1. Create a new Positron notebook
2. Add a code cell with `x = 1`
3. Add a markdown cell with `# Introduction` and `## Analysis` headers
4. Open the Outline pane (View > Outline or sidebar)
5. **Expected**: "Introduction" and "Analysis" appear in the outline with proper nesting (Analysis nested under Introduction). Code cell shows `x = 1` as preview.

### Scenario 2: Click-to-navigate from outline

1. With the notebook from Scenario 1 open
2. Click an outline entry (e.g., "Introduction")
3. **Expected**: The corresponding cell is selected and scrolled into view

### Scenario 3: Outline updates on cell content edit

1. Edit the markdown cell to change `# Introduction` to `# Overview`
2. **Expected**: After ~300ms, outline updates to show "Overview" instead of "Introduction"

### Scenario 4: Outline updates on cell add/remove

1. Add a new markdown cell with `# Conclusion`
2. **Expected**: Outline adds "Conclusion" entry
3. Delete the cell
4. **Expected**: Outline removes "Conclusion" entry

## Edge Cases

### Scenario 5: Multiple headers in one markdown cell

1. Create a markdown cell with multiple headers at different levels (h1, h2, h3)
2. **Expected**: All headers appear with correct nesting hierarchy

### Scenario 6: Empty notebook

1. Open a new empty notebook
2. **Expected**: Outline pane is empty or shows no entries

## Known v1 Limitations (Do NOT file bugs for these)

- No cursor-accurate heading tracking within multi-heading markdown cells (highlights first heading)
- No code cell symbols (functions, classes) -- only first-line preview
- No problem markers, toolbar actions, or configuration settings
